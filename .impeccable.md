# Design Context

## Users

開発者。自分のプロジェクトに組み込むUIコンポーネントを探している。
作業中のデスクで使う。コード例をすぐコピーして使いたい。
レジストリサイトに長居はしない — 必要なものを見つけて去る。

## Brand Personality

**精確・軽快・実用的**

- 精確: 曖昧さがない。余計なものがない。
- 軽快: "tyap" の音のように、リズムがある。反応が速い。
- 実用的: 見せるためのデザインではなく、使うためのデザイン。

## Aesthetic Direction

- **トーン**: Utilitarian with warmth. 工業的だが冷たくない。
- **テーマ**: ダーク/ライト両対応。開発者はダークモードが多いが、ドキュメントはライトも必要。
- **カラー**: Muted sage/olive green（hue ~155 OKLCH）をアクセントに。中性色はわずかにグリーン寄りに染める。
- **参照**: shadcn/ui のドキュメントサイト、Radix Themes、Tailwind CSS docs
- **反参照**: 派手なグラデーション、ネオン色、glassmorphism、過剰なアニメーション

## Design Principles

1. **即座に理解できる** — 説明不要のUI。来た瞬間にやることが分かる。
2. **密度と余白のバランス** — 情報密度を保ちつつ、呼吸できる余白を確保する。
3. **一貫性は信頼** — 同じものは同じ見た目。例外を作らない。
4. **装飾は意味を持つときだけ** — 意味のないビジュアルは存在しない。
5. **コードファースト** — このサイト自体がコンポーネントの実例である。
