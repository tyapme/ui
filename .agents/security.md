# セキュリティ

### 26. セキュリティ

#### 基本原則

- すべての外部入力は信用しない。
- サーバ側で必ずバリデーションする。
- 認可チェックはServer側で行う。

#### XSS対策

- `dangerouslySetInnerHTML` の使用は禁止。
- 外部HTMLをそのまま描画することは禁止。
- 文字列連結でHTMLを生成することは禁止。

#### 認証・認可

- 認証情報やトークンをClient Componentに直接渡す設計は禁止。
- roleや権限による表示制御をClientだけで完結させることは禁止。
- APIは常に認証・認可前提で実装する。

#### 環境変数

- APIキー・シークレットをコードに直接書くことは禁止。
- `NEXT_PUBLIC_` を安易に付けることは禁止。

#### API設計

- 外部I/Oは常に失敗前提で扱う。
- エラーメッセージに内部情報を含めることは禁止。

#### ログ

- 個人情報やトークンをログ出力することは禁止。
- 本番環境で不要なdebugログを出すことは禁止。
