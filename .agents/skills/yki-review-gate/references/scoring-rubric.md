# Scoring Rubric

## 合格ライン

- 合格: `96-100`
- 不合格: `0-95`

`95 点超え` が条件なので、`95` は通さない。

## 減点の考え方

### 100-98

- 指摘があっても軽微
- ルール違反なし
- 修正不要または任意改善のみ

### 97-96

- 軽い懸念はある
- ただし blocker はない
- 現時点で merge / push しても大きな事故は起きにくい

### 95-90

- 明確な修正要求がある
- ルール違反、設計不整合、境界条件漏れ、保守性問題のいずれかが残る
- コード変更に入る前に修正が必要

### 89 以下

- blocker が強い
- push / merge を止めるべき
- 重大バグ、重大な設計崩れ、重大なセキュリティ懸念、重大な保守性問題がある

## reviewer 共通の採点観点

- AGENTS.md と repo ルールを守っているか
- 既存アーキテクチャと整合しているか
- バグや境界条件に穴がないか
- セキュリティ事故の芽がないか
- 保守しやすいか
- 修正後に再レビューすべき理由が残っていないか

## 再レビューのルール

- 1 人でも `95 以下` なら全 reviewer を再実行する。
- 前回の指摘が 1 つでも残っているなら、通過扱いにしない。
- 点数だけ上がっても、根拠が弱ければ不合格扱いにする。
