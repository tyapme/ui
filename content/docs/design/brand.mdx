## ミッション

必要だから作る。使われるから続ける。

## What we are

個人開発者としてのブランド。設計・実装・運用を一人で行う。規模の小ささを弱点ではなく、意思決定の速さと品質への直接責任として示す。

## パーソナリティ

直接的 / 実用志向 / 静かな自信 / 誠実 / 技術的に正直

**使わない**: コーポレート的な表現、過剰なフレンドリーさ、スタートアップ風の誇大表現

## ボイス & トーン

| 項目 | ルール |
| --- | --- |
| 言葉遣い | 平易・簡潔。専門用語は必要なときだけ使い、使ったら説明する |
| 一人称 | 使わない。主語を省くか、プロダクト名を主語にする |
| エラー文 | 謝罪より原因と次の行動。「〜が失敗しました。〜を確認してください」 |
| 完了文 | 達成感ではなく事実報告。「保存しました」「送信完了」 |
| 禁止 | 「素晴らしい」などの過剰な肯定。絵文字の乱用 |
