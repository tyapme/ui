## 選択日付（タイムゾーン対応）

`Calendar` コンポーネントは `timeZone` プロパティを受け付け、ユーザーのローカルタイムゾーンで日付を表示・選択するために利用します。

```tsx showLineNumbers
export function CalendarWithTimezone() {
  const [date, setDate] = React.useState<Date | undefined>(undefined)
  const [timeZone, setTimeZone] = React.useState<string | undefined>(undefined)

  React.useEffect(() => {
    setTimeZone(Intl.DateTimeFormat().resolvedOptions().timeZone)
  }, [])

  return (
    <Calendar
      mode="single"
      selected={date}
      onSelect={setDate}
      timeZone={timeZone}
    />
  )
}
```

**注意:** 選択日付にずれが発生する場合（例: 20日を選ぶと 19日がハイライトされる）は、`timeZone` プロパティがユーザーのローカルタイムゾーンに設定されているか確認してください。

**なぜクライアントサイドなのか?** タイムゾーンは `useEffect` 内で `Intl.DateTimeFormat().resolvedOptions().timeZone` を用いて検出されます。サーバーとクライアントのタイムゾーンが異なる場合にハイドレーションの不一致が発生するため、レンダリング中に検出することは避けています。

```tsx showLineNumbers

  selected={date}
  onSelect={setDate}
  className="rounded-lg border [--cell-size:--spacing(11)] md:[--cell-size:--spacing(12)]"
/>
```

または固定値を指定することもできます:

```tsx showLineNumbers
<Calendar
  mode="single"
  selected={date}

  className="rounded-lg border [--cell-size:2.75rem] md:[--cell-size:3rem]"
/>
```

RTL を使用する場合は、`react-day-picker/locale` からロケールをインポートし、`locale` と `dir` の両方の props を Calendar コンポーネントに渡します:

```tsx showLineNumbers
import { arSA } from "react-day-picker/locale"

;<Calendar
  mode="single"
  selected={date}
  onSelect={setDate}
  locale={arSA}
  dir="rtl"
/>
```

## 変更履歴

### RTL 対応

`Calendar` コンポーネントの以前のバージョンからアップグレードする場合、ロケールサポートを追加するために以下の変更を適用する必要があります:

<Steps>

<Step>

`Locale` 型をインポートします。

`react-day-picker` のインポートに `Locale` を追加します:

```diff
  import {
    DayPicker,
    getDefaultClassNames,
    type DayButton,
+   type Locale,
  } from "react-day-picker"
```

</Step>

<Step>

Calendar コンポーネントに `locale` prop を追加します。

Props に `locale` を追加します:

```diff
  function Calendar({
    className,
    classNames,
    showOutsideDays = true,
    captionLayout = "label",
    buttonVariant = "ghost",
+   locale,
    formatters,
    components,
    ...props
  }: React.ComponentProps<typeof DayPicker> & {
    buttonVariant?: React.ComponentProps<typeof Button>["variant"]
  }) {
```

</Step>

<Step>

`DayPicker` に `locale` を渡します。

`DayPicker` コンポーネントに `locale` prop を渡します:

```diff
    <DayPicker
      showOutsideDays={showOutsideDays}
      className={cn(...)}
      captionLayout={captionLayout}
+     locale={locale}
      formatters={{
        formatMonthDropdown: (date) =>
-         date.toLocaleString("default", { month: "short" }),
+         date.toLocaleString(locale?.code, { month: "short" }),
        ...formatters,
      }}
```

</Step>

<Step>

`CalendarDayButton` が locale を受け入れるように更新します。

`CalendarDayButton` コンポーネントのシグネチャーを更新し、`locale` を渡します:

```diff
  function CalendarDayButton({
    className,
    day,
    modifiers,
+   locale,
    ...props
- }: React.ComponentProps<typeof DayButton>) {
+ }: React.ComponentProps<typeof DayButton> & { locale?: Partial<Locale> }) {
```

</Step>

<Step>

`CalendarDayButton` の日付フォーマットを更新します。

日付フォーマットで `locale?.code` を使用します:

```diff
    <Button
      variant="ghost"
      size="icon"
-     data-day={day.date.toLocaleDateString()}
+     data-day={day.date.toLocaleDateString(locale?.code)}
      ...
    />
```

</Step>

<Step>

`DayButton` コンポーネントに locale を渡します。

`DayButton` の使用箇所を更新して `locale` prop を渡します:

```diff
      components={{
        ...
-       DayButton: CalendarDayButton,
+       DayButton: ({ ...props }) => (
+         <CalendarDayButton locale={locale} {...props} />
+       ),
        ...
      }}
```

</Step>

<Step>

RTL 対応の CSS クラスを更新します。

方向指定クラスを論理プロパティに置き換えてより良い RTL 対応を実現します:

```diff
  // In the day classNames:
- [&:last-child[data-selected=true]_button]:rounded-r-(--cell-radius)
+ [&:last-child[data-selected=true]_button]:rounded-e-(--cell-radius)
- [&:nth-child(2)[data-selected=true]_button]:rounded-l-(--cell-radius)
+ [&:nth-child(2)[data-selected=true]_button]:rounded-s-(--cell-radius)
- [&:first-child[data-selected=true]_button]:rounded-l-(--cell-radius)
+ [&:first-child[data-selected=true]_button]:rounded-s-(--cell-radius)

  // In range_start classNames:
- rounded-l-(--cell-radius) ... after:right-0
+ rounded-s-(--cell-radius) ... after:end-0

  // In range_end classNames:
- rounded-r-(--cell-radius) ... after:left-0
+ rounded-e-(--cell-radius) ... after:start-0

  // In CalendarDayButton className:
- data-[range-end=true]:rounded-r-(--cell-radius)
+ data-[range-end=true]:rounded-e-(--cell-radius)
- data-[range-start=true]:rounded-l-(--cell-radius)
+ data-[range-start=true]:rounded-s-(--cell-radius)
```

</Step>

</Steps>

これらの変更を適用すると、`locale` prop でロケール固有のフォーマットを提供できます:

```tsx
import { enUS } from "react-day-picker/locale"

;<Calendar mode="single" selected={date} onSelect={setDate} locale={enUS} />
```
